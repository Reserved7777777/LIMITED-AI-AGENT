## 架构知识积累
### Gateway 重启
- `gateway restart` 有 coalescing 机制，多次调用会卡住
- 安全方式：`systemctl --user restart openclaw-gateway.service`（延迟 1 分钟）

### Nginx
- WebSocket proxy 的 `Connection: upgrade` 不能写死，要用 `map` 变量
- `X-Forwarded-Proto: $scheme` 必须传，否则 WSS URL 生成 `ws://` 而非 `wss://`
- `gateway.trustedProxies` 必须配 `["127.0.0.1", "::1"]`

### Config 覆写保护
- Gateway 检测 JSON 被外部修改 → 创建 `.clobbered.*` 备份并恢复缓存版本
- 修改方法：停 Gateway → 改 JSON → 删 `.clobbered.*` 和 `.last-good`
- 安全路径：`openclaw config set` CLI

## 架构知识积累
### Gateway 重启
- `gateway restart` 有 coalescing 机制，多次调用会卡住
- 安全方式：`systemctl --user restart openclaw-gateway.service`（延迟 1 分钟）

### Nginx
- WebSocket proxy 的 `Connection: upgrade` 不能写死，要用 `map` 变量
- `X-Forwarded-Proto: $scheme` 必须传，否则 WSS URL 生成 `ws://` 而非 `wss://`
- `gateway.trustedProxies` 必须配 `["127.0.0.1", "::1"]`

### Config 覆写保护
- Gateway 检测 JSON 被外部修改 → 创建 `.clobbered.*` 备份并恢复缓存版本
- 修改方法：停 Gateway → 改 JSON → 删 `.clobbered.*` 和 `.last-good`
- 安全路径：`openclaw config set` CLI

**v1（20:13）**：build_report.py 改用 marker 边界替换（`<!-- ===== RISK ===== -->` → `<!-- ===== EVENTS ===== -->`），不再依赖 `_replace_tag`。✅ build_report 侧搞定。

**v2（21:08）**：update_realtime.py 的 `sync_analysis()` 吞掉风险 sec-label。根因：`risk_start = html.find('<div class="risk-list">', cfx_end)` 找到的是 `<div class="risk-list">`，但 `<!-- ===== RISK ===== -->` 注释和 sec-label 在这个位置**之前**，analysis 替换区间（`html[:cfx_end] + new_html + html[risk_start:]`）把它们当作"中间内容"吃掉了。

修复：`sync_analysis()` 主路径和 fallback 都改用 `<!-- ===== RISK ===== -->` 作为 risk_start 边界；`sync_risks_from_cls()` 的 `risk_html` 改为包含完整注释+sec-label。

**v3（21:24）**：auto-repair 覆盖已打好的商品价格。根因：SELF-CHECK（步骤9）在 commodity patch（步骤1-5）之后执行，auto-repair 调用 `build_report.py` 重建 HTML，使用的是缓存旧数据，覆盖了刚刚从 futunn 抓取的最新价格。

修复：SELF-CHECK 移到 main() 开头、所有修改之前执行。这会稍微增加每次运行的延迟（auto-repair 场景），但确保 patch 始终在 clean HTML 上运行。

**永久方案**：SELF-CHECK 检查 13 个标记 → 缺失就 auto-repair → commodity patch 跑最新数据 → risk sync → analysis sync → write。任何标记完整性问题都在修改前修复，修改后不再有二次覆盖。

### USD/CNY 精度丢失

商品卡 USD/CNY 显示 `6.81`（2位小数），实际应为 `6.8134`（4位小数）。`patch_html()` 对所有商品统一使用 `f"{p:,.2f}"`。修复：`patch_html` 新增 `price_fmt` 参数（默认 `:,.2f`），调用 USD/CNY 时传 `:,.4f`。
