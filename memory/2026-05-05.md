## 架构原则（2026-05-05 17:49 用户指定）

### 核心关系
```
MARCUS 动量分析报告
           ↓   呈现/翻译
   HTML 模板（固定构件框架）
           ↓   填入
   实时数据（价格/涨跌/标签）
```

**HTML 是 MARCUS 报告的翻译呈现层，不是独立页面。**

### 构件分层
- **固定构件**（写死，不删不改）：指数卡、商品卡、立场条、表格结构、日历网格、风险列表、样式定义
- **动态数据**（定时更新）：价格、涨跌幅、成交量、市值、涨跌色、催化标签

### 生成规则
1. 每日首次：`fetch_market_data` → JSON → `build_report` → HTML（与锚点一致）
2. 每30分钟：`update_realtime` → 替换 5 个实时价格，不动结构
3. 任何新生成的 HTML 必须与当前锚点结构逐项比对

## 错误日志: VIX patch 误写上证指数（2026-05-05 18:25-18:40）

**现象：** `update_realtime.py` 每30分钟运行时，上证指数卡片价格被错误替换为 VIX 最新值（如 19.50）。

**根因：** `patch_html` 函数搜索 marker "VIX" 后的 regex 未限定范围。VIX 卡的 price div 格式特殊（`<div class="price" style="color:var(--red);">`，无 up/down class），regex `price\s+(up|down)` 不匹配后继续往后搜，匹配到紧接着的下一个同格式卡片——上证指数。

**修复（18:40）：**
1. `patch_html` 增加搜索边界 `</a>`，防止跨卡匹配
2. VIX 单独处理：直接 regex 匹配 `class="price" style="color:var(--red);">` 格式

**验证：** 修复后 8 张指数卡 + 5 张商品卡数据完全正确。

## 每日动量报告 重大更新

### HTML 报告架构
- **域名**: `limitedaiagent.icu` → nginx → `/var/www/openclaw/`
- **模板**: `report-template.html` + `build_report.py`（MARCUS AI 生成）
- **30分钟实时更新**: `update_realtime.py`（cron `e84f5b9f` → `*/30 7-23 * * 1-5`）
- **JSON 数据**: `/tmp/report-data-YYYY-MM-DD.json`

### 30分钟更新机制（fix_all 一次性方案）
`update_realtime.py` 是一个独立脚本，在30分钟cron里全量运行：
1. `sync_prices` — 爬 commodity/cfx 价格（港股美股周末休市）
2. `sync_risks_from_cls` — 爬财联社电报 + URL验证
3. `sync_events` — 经济日历（目前固定8项）
4. `sync_analysis` — 商品汇率分析（基于价格阈值规则）
5. `sync_stance` — 立场条（数据驱动：SH>4120→保守买入50%, VIX↑→保守空头等）
6. `patch_html` — 原地替换风险区块（新旧都已实现）和价格卡片

**关键函数**：
- `sync_risks_from_cls`: Playwright 爬 `cls.cn/telegraph` → 提取【】标题 → 找最近的 `/detail/{id}` → HTTP验证 → 生成HTML区块
- `extract_price`（`sync_analysis`内）: 正则搜索 `</a>` 边界内找价格，防跨卡片污染
- `extract_val_chg`（`sync_stance`内）: 同上边界
- `patch_html`: 在HTML中定位 `<!-- ===== RISK ===== -->` / `<!-- ===== COMMODITY ===== -->` 等标记并替换
